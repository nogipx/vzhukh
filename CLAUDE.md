## Правила общения с разработчиком.

- отвечай всегда кратко и по существу.
- отвечай всегда на английском для экономии токенов.
- критически анализируй вопросы и предложения разработчика.
- помогай разработчику в теории.
- не генерируй код которого не просили явно.

## Где лежит код?

- все исходники с которыми мы работаем лежат в ./lib/src
- все тесты с которыми мы работаем лежат в ./test
- для изучения используемых библиотек ищи либо по пути в ./pubspec.yaml, либо в ~/.pub-cache

## Как запускать код/тесты?

- всегда используй fvm dart для запуска кода

## Где лежат скрипты?

- для создания скриптов надо использовать ./Justfile

## Code Style

- все юзкейсы должны быть callable dart классами
